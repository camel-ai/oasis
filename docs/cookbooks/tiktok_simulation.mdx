# Short-Video Simulation

This cookbook shows how to run a short-video social simulation in OASIS using the TikTok-style platform preset. The focus is on social behavior: video creation, feed consumption, recommendation dynamics, and livestream interaction.

## What This Simulation Covers

- Short-video uploads and feed browsing
- Traffic-pool-style recommendation
- Watch-completion and negative-feedback signals
- Creator-viewer interactions
- Livestream entry, exit, comments, and gifts

## Minimal Example

```python
import asyncio
import os
import tempfile

from camel.models import ModelFactory
from camel.types import ModelPlatformType, ModelType

import oasis
from oasis import (ActionType, AgentGraph, LLMAction, ManualAction,
                   SocialAgent, UserInfo)


async def main():
    model = ModelFactory.create(
        model_platform=ModelPlatformType.OPENAI,
        model_type=ModelType.GPT_4O_MINI,
    )

    available_actions = ActionType.get_default_tiktok_actions()
    agent_graph = AgentGraph()

    creator = SocialAgent(
        agent_id=0,
        user_info=UserInfo(
            user_name="dance_queen",
            name="Lily",
            description="Dance creator [interests: dance, music, trending]",
            profile={
                "other_info": {
                    "user_profile": "Professional dancer sharing challenges",
                    "role": "creator",
                }
            },
            recsys_type="tiktok",
        ),
        agent_graph=agent_graph,
        model=model,
        available_actions=available_actions,
    )
    agent_graph.add_agent(creator)

    viewer = SocialAgent(
        agent_id=1,
        user_info=UserInfo(
            user_name="music_fan",
            name="Tom",
            description="Casual viewer [interests: dance, comedy, music]",
            profile={
                "other_info": {
                    "user_profile": "Short-video user who scrolls daily",
                    "role": "viewer",
                }
            },
            recsys_type="tiktok",
        ),
        agent_graph=agent_graph,
        model=model,
        available_actions=available_actions,
    )
    agent_graph.add_agent(viewer)

    db_path = os.path.join(tempfile.gettempdir(), "tiktok_example.db")
    if os.path.exists(db_path):
        os.remove(db_path)

    env = oasis.make(
        agent_graph=agent_graph,
        platform=oasis.DefaultPlatformType.TIKTOK,
        database_path=db_path,
        tiktok_recsys_params={
            "score_weights": {
                "completion_rate": 0.35,
                "like_rate": 0.15,
                "comment_rate": 0.15,
                "share_rate": 0.20,
                "negative_rate": 0.15,
            },
            "promote_percentile": 0.20,
            "demote_percentile": 0.70,
            "max_pool_level": 7,
            "rec_mix": {
                "interest": 0.70,
                "following": 0.15,
                "explore": 0.10,
            },
            "decay_half_life": 72,
        },
    )

    await env.reset()

    await env.step({
        creator: ManualAction(
            action_type=ActionType.UPLOAD_VIDEO,
            action_args={
                "content": "Amazing dance challenge #dance #trending",
                "duration_seconds": 15,
                "category": "dance",
                "topic_tags": '["dance", "challenge", "trending"]',
                "quality_score": 0.85,
                "hook_strength": 0.90,
            },
        )
    })

    await env.step({viewer: LLMAction()})

    await env.step({
        viewer: ManualAction(
            action_type=ActionType.WATCH_VIDEO,
            action_args={"post_id": 1, "watch_ratio": 0.85},
        )
    })

    await env.step({
        creator: LLMAction(),
        viewer: LLMAction(),
    })

    await env.close()


if __name__ == "__main__":
    asyncio.run(main())
```

## Recommended Actions

For short-video simulations, the default action set includes:

- `UPLOAD_VIDEO`
- `WATCH_VIDEO`
- `LIKE_POST`
- `CREATE_COMMENT`
- `SHARE_VIDEO`
- `FOLLOW`
- `DUET`
- `STITCH`
- `NOT_INTERESTED`
- `ENTER_LIVESTREAM`
- `EXIT_LIVESTREAM`
- `LIVESTREAM_COMMENT`
- `SEND_GIFT`
- `START_LIVESTREAM`
- `END_LIVESTREAM`
- `SEARCH_POSTS`
- `TREND`
- `DO_NOTHING`

## Recommendation Parameters

The TikTok-style preset supports configurable recommendation behavior through `tiktok_recsys_params`.

Common controls:

- `score_weights`: Weights for completion, likes, comments, shares, and negative feedback
- `promote_percentile`: Top share of videos promoted to the next traffic pool
- `demote_percentile`: Bottom share of videos demoted out of active distribution
- `max_pool_level`: Maximum traffic-pool tier
- `rec_mix`: Blend of interest-based, following-based, and exploration traffic
- `decay_half_life`: Time-decay strength for older videos

## Key Metrics To Track

For short-video social simulations, these metrics are especially useful:

- `view_count`: How often videos are actually watched
- `avg_watch_ratio`: How much of each video viewers complete on average
- `share_count`: A strong signal for creator spread and downstream reach
- `negative_count`: Explicit rejection feedback from viewers
- `traffic_pool_level`: Which distribution tier a video currently occupies
- `current_viewers` / `peak_viewers`: Useful for livestream attention dynamics
- `total_comments` / `total_gifts_value`: Useful for livestream interaction intensity

## Paper-Style Experiment Template

If you want a reproducible counterfactual experiment instead of a free-form simulation, OASIS also includes a deterministic short-video template:

```bash
python examples/experiment/short_video_negative_feedback/run_experiment.py
```

This experiment creates:

- a `baseline` run with limited negative feedback on a low-quality video
- a `treatment` run with stronger `not_interested` signals on that same video

The outputs are written to:

```text
examples/experiment/short_video_negative_feedback/output
```

Then you can analyze the two runs with:

```bash
python visualization/short_video_simulation/code/compare_runs.py \
  baseline=examples/experiment/short_video_negative_feedback/output/negative_feedback_baseline.db \
  treatment=examples/experiment/short_video_negative_feedback/output/negative_feedback_treatment.db \
  --output-dir examples/experiment/short_video_negative_feedback/output/comparison
```

## Built-In Observability Helper

OASIS also provides a built-in helper for summarizing short-video simulations directly from the SQLite database:

```python
from oasis import get_short_video_observability_report

report = get_short_video_observability_report("./data/tiktok_simulation.db")

print(report["summary"])
print(report["creators"][:3])
print(report["top_videos"][:5])
print(report["livestreams"][:3])
```

The returned report includes:

- `summary`: Overall feed-level metrics
- `creators`: Creator growth and engagement summaries
- `top_videos`: Video-level ranking and engagement observations
- `livestreams`: Livestream attention and interaction summaries

For time-series analysis, you can also use:

```python
from oasis import get_short_video_time_series_report

time_series = get_short_video_time_series_report("./data/tiktok_simulation.db")

print(time_series["video_time_series"])
print(time_series["creator_growth"])
print(time_series["livestream_time_series"])
print(time_series["viewer_retention_time_series"])
```

## Visualization Script

OASIS now includes a lightweight visualization helper for short-video simulations:

```bash
python visualization/short_video_simulation/code/plot_metrics.py \
  ./data/tiktok_simulation.db \
  --output-dir ./visualization/short_video_simulation/output
```

This script exports CSV summaries and generates PNG charts for:

- feed growth and watch dynamics
- traffic-pool movement
- negative-feedback accumulation
- livestream attention and viewer retention

## Markdown Research Report

To generate a compact research-style markdown summary from a simulation database:

```bash
python visualization/short_video_simulation/code/generate_report.py \
  ./data/tiktok_simulation.db \
  --output ./visualization/short_video_simulation/output/report.md
```

This report includes:

- top creators
- top videos
- livestream summary
- latest feed-step snapshot

## Comparing Multiple Runs

To compare multiple experimental databases, use:

```bash
python visualization/short_video_simulation/code/compare_runs.py \
  baseline=./data/tiktok_baseline.db \
  experiment=./data/tiktok_experiment.db \
  --output-dir ./visualization/short_video_simulation/output/comparison
```

This comparison script exports a combined time-series CSV and a PNG chart that contrasts:

- cumulative views
- average watch ratio
- average traffic-pool level
